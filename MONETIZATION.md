# ScoreBooth 수익화 정책 — 리스크 판정 기준

이 문서는 어떤 기능을 만들지 말지, 유료로 해도 되는지 판단할 때 참고하는 기준 문서입니다. 새 기능을 추가하기 전에 여기 원칙에 먼저 맞춰보고 결정합니다.

## 한 줄 원칙

- **MLB가 만든 데이터 = 항상 무료.** 예외 없음.
- **사용자 본인이 만든 데이터(기록/메모/큐레이션/그룹) = 유료화해도 안전.**
- **"구독"이라 부르면 위험, "후원자 등급/보너스"라 부르면 안전** — 같은 기능이라도 프레이밍이 리스크를 가른다.
- **콘텐츠를 직접 호스팅하면 위험, 링크/공식 임베드로 걸면 안전.**
- 알림은 유일한 회색지대라 별도 취급.

배경: ScoreBooth는 MLB의 비공식·무라이선스 Stats API와 저작권 있는 팀 로고/데이터를 사용한다. README에 "개인적·비상업적 용도"라고 명시했고, 이 선을 넘으면(=MLB IP를 이용해 실제로 돈을 버는 구조가 되면) MLB Advanced Media 쪽에서 문제 삼을 법적 리스크가 커진다. 아래 판정은 전부 이 전제에서 나온다.

## 실행 메모

- 유료 등급은 별도 결제 시스템을 새로 만들지 않고 **Ko-fi Membership Tiers**를 그대로 사용한다. 무료 플랜에서도 3개 티어까지 가능하고, 일회성 팁은 수수료 0%, 정기 멤버십은 5%(Ko-fi Gold $12/월이면 0%).
- 새 기능을 추가하기 전에 이 표에 없는 항목이면, "MLB가 만든 데이터인가 / 사용자 본인 데이터인가"부터 먼저 따져서 위 원칙에 맞춰 분류하고 이 표에 추가한다.
